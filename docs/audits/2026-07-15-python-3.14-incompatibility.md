# Python 3.14 Incompatibility

## Resolution (final, 2026-07-15)

Reverted to Python 3.11 rather than continuing to patch individual Python
3.14 occurrences. Three pre-existing monkeypatches in `claudia/app.py`
(added 2026-06-09, commit `59a7bd90`) were removed entirely — see
`docs/plans/2026-07-15-python-3.11-pin.md`. A claimed 4th occurrence
(`IBKRWebSocket.connect()`) did not reproduce under direct testing of the
real invocation path; the actual `ExecutionListener` failures were traced
to an unrelated URL-doubling bug in `ibkr_core_mcp/streaming.py`, since
fixed and committed independently of the Python-version decision.

Rationale: the three existing patches reached into `anyio`/`asyncio`
private internals with no compatibility guarantee, on a codebase that
stages real trades. The full scope of Python 3.14 breakage was never
mapped (only the boot-level path was ever tested) — patching individual
occurrences as they surfaced was open-ended effort with no clear end,
against a Python version with no functional benefit to ClaudIA today.
3.11 is what `pyproject.toml`, `ruff`, and `ibkr_core_mcp`'s own
classifiers already declare as designed-for.

A UI-framework migration away from Chainlit (to NiceGUI) remains a
separate, deliberate future initiative — unrelated to this decision.
